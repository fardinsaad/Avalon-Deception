# Avalon Deception Dataset - Dialogue Generation & Verification System

## Overview

This repository contains a complete pipeline for generating and verifying AI-generated Avalon Round 1 dialogues. The system uses Large Language Models (LLMs) to generate realistic game discussions, then employs **Claude Sonnet 4.5** as an independent verifier to evaluate and select the best outputs using **5 binary pass/fail criteria**.

## Game Background

**The Resistance: Avalon** is a social deception game where:
- **5 players** split into **Good** (3 players) and **Evil** (2 players) teams
- Players discuss quest outcomes to identify hidden roles
- Good players use transparent, cooperative tactics
- Evil players employ deception to avoid detection

### What Gets Generated

For each game, the system generates:

1. **discussion_log**: 4-player dialogues following the format:
   ```
   Discussion after Quest 1:
   P2: "I think the quest failure was suspicious..."
   P3: "We need to analyze the voting patterns..."
   P4: "I trust this team's composition."
   P5: "Let's be cautious about who we trust."
   ```

2. **matrix_tactic_scale**: JSON mapping each player's tactics to the 4×4 behavior matrix

### The 4×4 Behavior Matrix

The system uses a research-based behavior matrix with **37 tactical patterns**:

**Rows (Information Strategy):**
- Transparent
- Selective/Framing
- Careless-to-truth
- Counterfactual

**Columns (Goal Orientation):**
- Cooperative
- Defensive
- Opportunistic
- Adversarial

**Example tactics:**
- Evidence sharing (Good players - Transparent × Cooperative)
- Hard lying (Evil players - Counterfactual × Adversarial)
- Deflection (Both - various combinations)

**Note**: Verification is much faster than generation because it only compares existing outputs rather than generating new content.

### Binary Multi-Criteria Validation

Claude Sonnet 4.5 evaluates each dialogue pair using **5 binary pass/fail checks**:

1. **Player Roles Consistency** (✓/✗) - Good use honest tactics, Evil use deceptive tactics
2. **Public History Alignment** (✓/✗) - Dialogue fits game state
3. **Matrix Tactic Scale Validity** (✓/✗) - Correct tactical classification
4. **Avalon Gameplay Authenticity** (✓/✗) - Natural, strategic dialogue
5. **Dialogue Format Compliance** (✓/✗) - Exactly 4 speakers, correct format

### Decision Logic

Based on total scores (X/5), Claude applies this logic:

1. **Both 5/5** → Pairwise comparison to select the better dialogue
2. **One 5/5, other <5/5** → Select the 5/5 response
3. **At least one 4/5** → Apply **Targeted Correction** (fix only the failing criterion)
   - If both 4/5, default to GPT-5.2
4. **Both ≤3/5** → **Full Custom Generation** (generate new dialogue from scratch)

### Output Files

#### 1. verified_r1_seeds_combined.csv

Selected/corrected dialogues with metadata:
- All original columns from input datasets
- `LLM_used`: Source (Gemini-3.1, GPT-5.2, Gemini-3.1-Claude-4.5, GPT-5.2-Claude-4.5, or Claude-4.5)
- `Claude_Reasoning`: Explanation for the selection decision

#### 2. verified_r1_criteria_scores.csv

Per-criterion validation tracking:
- `ID`: Game/Round identifier
- `Gemini31_Roles`, `Gemini31_History`, `Gemini31_Matrix`, `Gemini31_Authenticity`, `Gemini31_Format`: Binary (1=pass, 0=fail)
- `Gemini31_Total`: Score (e.g., "5 of 5")
- `GPT52_*`: Same for GPT-5.2
- `Selected_LLM`: Chosen model
- `Correction_Mode`: None, Pairwise_Tiebreaker, Targeted, or Full_Custom

## Troubleshooting

### Common Issues

**API Key Errors**:
- Verify `.env` file exists with correct keys
- Run `python llm.py` or `python gemini.py` to test connections

**Rate Limiting**:
- Generation includes 1-second delays between calls
- Verification includes 1-second delays
- Adjust delays in cells if needed

**Memory Issues**:
- Close other Jupyter notebooks
- Restart kernel and clear outputs: Kernel → Restart & Clear Output

**Missing Files**:
- Ensure `Deception-Dataset.csv` exists before running log-gen.ipynb
- Generate both CSV files before running verification
